Ви — експерт з UI/UX дизайну, що спеціалізується на іконографії для професійного програмного забезпечення.

**Ваше завдання:** Створити повний набір SVG іконок для веб-додатку, що є аналогом CAD-програми. Для кожного імені зі списку нижче, ви повинні згенерувати один SVG-файл, що ідеально відповідає наступній **майстер-специфікації**.

### Майстер-специфікація

**1. Основна філософія:**
*   **Чіткість та мінімалізм:** Кожна іконка повинна бути миттєво впізнаваною.
*   **Професійна точність:** Використовуйте чисті, геометрично правильні форми.

**2. Технічні вимоги (не підлягають обговоренню):**
*   **Формат:** SVG.
*   **Розмір:** Дизайн повинен бути створений для `viewBox="0 0 24 24"`.
*   **Колір:** **Критично важливо!** Усі елементи `<path>` повинні використовувати `fill="currentColor"`. Не використовуйте шістнадцяткові коди кольорів (наприклад, `#FFFFFF` або `#000000`). Це дозволяє змінювати колір іконки через CSS.
*   **Фон:** Прозорий.
*   **Код:** SVG-код має бути максимально чистим і мінімальним. Використовуйте лише елемент `<path>`. Жодних метаданих, коментарів, ID, класів або `<style>` блоків.

**3. Керівництво по візуальному стилю:**
*   **Стиль:** Контурний (line art / outline). Усі іконки мають виглядати як єдиний набір.
*   **Товщина ліній:** Використовуйте послідовну товщину обведення приблизно **1.5px** (відносно `viewBox="0 0 24 24"`).
*   **Кути:** Переважно гострі. Якщо кути заокруглені, використовуйте послідовний, дуже малий радіус (наприклад, 1px).
*   **Деталізація:** Мінімальна. Жодних градієнтів, тіней, 3D-ефектів або декоративних елементів, що не несуть функціонального навантаження.
*   **Відступи:** Іконка повинна мати внутрішні відступи, займаючи близько 80-90% полотна, щоб не торкатися країв.

**4. Концептуальні метафори:**
*   Використовуйте універсально зрозумілі метафори для CAD та файлових операцій.
*   Приклад: "Обрізати" (Trim) — лезо ножиць, що перерізає лінію. "Дзеркало" (Mirror) — об'єкт і його симетричне відображення через центральну вісь.

**5. Формат виводу:**
Надайте відповідь у вигляді списку, де кожен елемент містить назву іконки, а за нею — блок коду з повним, чистим SVG.
